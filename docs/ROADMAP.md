# Roadmap

The roadmap uses implementation gates instead of launch promises.

## Current RC Candidate

- Converge the member product on Chat, Workspace, Apps, and Account.
- Preserve account ownership across conversations, tasks, memory, artifacts, and history.
- Complete App lifecycle and readiness states.
- Return capability results and evidence to the active workspace.
- Keep wallet and exchange work read-only unless a separate approval boundary is validated.

## Next Validation Gate

- Produce a clean candidate dependency install, lint, build, and local gate run.
- Promote only to the RC environment through a verified release path.
- Run authenticated browser validation for first-use, refresh, reconnect, and account isolation.
- Capture current product screenshots from the promoted candidate.
- Validate connection-dependent Apps under real account conditions.

## Product Completion Gate

- Recommend a suitable solution from an open-ended user goal.
- Create or select the correct workspace for that solution.
- Install and configure the required Apps with minimal setup.
- Keep every step understandable in user language.
- Preserve results, reusable context, and execution history.

## Public Promotion Gate

- Confirm product parity between the promoted RC candidate and the intended public site.
- Replace public copy, screenshots, and demo instructions only after parity is verified.
- Publish reproducible performance or reliability metrics only when benchmark artifacts exist.

## Research Direction

Embodied AI and GO2 integration remain separate research work. They are not part of the current TRACE product-completion claim.
