# Why TRACE

## AI Adoption Is More Than Model Access

Giving someone a chat box does not solve the operational work around AI. A useful system also needs to know which data it may use, what services are connected, where the work belongs, what requires approval, what result was produced, and how the user can continue later.

## The Integration Burden Is Usually Pushed To The User

Many AI products ask users to choose models, configure tools, understand agents, manage prompts, and stitch results together across applications. That creates friction before useful work begins.

TRACE takes the opposite approach:

- Begin with the goal.
- Keep technical setup out of the first interaction.
- Request a connection only when the task needs it.
- Preserve the result in the same work context.
- Make uncertainty and blockers visible.

## Why A Workspace Matters

Conversation is a good entry point, but it is not enough for multi-step work. Workspace gives the conversation a durable place for tasks, artifacts, memory, and history. It also makes it possible to stop, review, and resume without reconstructing the entire context.

## Why Apps Matter

Apps turn infrastructure capabilities into recognizable work. A user should see document analysis, research, Gmail work, market analysis, or project monitoring, not a registry of internal adapters.

## Why Reviewability Matters

AI systems can be useful without pretending to be autonomous. TRACE records results, surfaces missing setup, and keeps consequential actions behind explicit approval. This makes the system easier to trust, debug, and improve.
